# Qiita記事③ アウトライン

## タイトル
**非エンジニアがClaude Codeで年収を上げるロードマップ2026年版**

---

## 1. 記事の目的

- 「コードは書けないが、業務の課題は誰よりも知っている」非エンジニア人材が、Claude Code を相棒に **自分の市場価値（＝年収）を上げる** ための具体的な行動指針を提示する。
- 抽象論ではなく、**2026年4月時点で実際に使えるツール・案件単価・学習順序** を、筆者自身（HR出身→Claude CodeでBtoB SaaSを実装中）の体験ベースで書く。
- 「AIに仕事を奪われる側」ではなく「AIで仕事を作る側」に回るための実装可能なロードマップを残す。

---

## 2. 各セクションの構成と要点

### 2.1 はじめに：なぜ今「非エンジニア × Claude Code」なのか
- 2026年現在、コード生成の精度は "ジュニアエンジニア相当" を超えてシニア帯に入った
- ボトルネックは「書く力」ではなく **「要件を言語化する力」「ドメイン知識」「最後までやり切る力」**
- これらを持つ非エンジニアこそ、Claude Code 時代の最大の受益者である

### 2.2 結論先出し：年収UPの3ルート
1. **社内DXルート**: 今の職場の業務を Claude Code で自動化し、評価と昇給につなげる
2. **副業ルート**: 中小企業の "業務効率化案件" を月10〜30万で受託する
3. **プロダクト所有ルート**: 自分で SaaS を立てて MRR を積む（本記事筆者はこのルート）
- それぞれの想定年収レンジ・必要期間・難易度を表で提示

### 2.3 ロードマップ Phase 1（0〜1ヶ月）：環境を整える
- Claude Code のインストールと最低限の設定（`CLAUDE.md`, MCP, settings.json）
- Git / GitHub の "怖くない" 触り方（`gh` CLI を Claude に任せる前提でOK）
- 最初の壁は "ターミナルアレルギー" — それを越える練習メニュー

### 2.4 ロードマップ Phase 2（1〜3ヶ月）：自分の業務を1つ自動化する
- 題材は **必ず "自分が今週困っていること"** から選ぶ
- 例: Excel集計、議事録要約、問い合わせ一次返信、定型レポート生成
- 完成物を Loom などで録画 → 社内で配る → 評価面談で持ち出す

### 2.5 ロードマップ Phase 3（3〜6ヶ月）：成果物を "外向き" にする
- GitHub に Public リポジトリを作り、README を整える（Claude Code に書かせて良い）
- ポートフォリオサイト1枚（Next.js + Vercel で十分）
- Qiita / Zenn で **"非エンジニアが作った" という属性を隠さず** 1記事書く（隠すより刺さる）

### 2.6 ロードマップ Phase 4（6〜12ヶ月）：マネタイズ
- 副業ルート: ランサーズ／YOUTRUST／X の DM 経由で月1案件取る具体手順
- プロダクトルート: Stripe を繋いで $10/mo でも良いから "課金された経験" を作る
- 社内ルート: 評価制度に "AI活用実績" 項目があるかを確認し、無ければ作る提案をする

### 2.7 やってはいけないこと
- 「Python を1から学び直す」← 最短ではない、Claude Code に書かせて読めれば十分
- 「資格を取ってから動く」← Phase 2 の自動化を1個出した方が100倍評価される
- 「AIで作りました」を隠す ← バレる、信用を失う、最初から開示した方が応援される

### 2.8 ドメイン知識こそ最大の武器
- HR、経理、医療、物流、製造、教育 — それぞれの "現場の細かい異常系" は AI が知らない
- 自分の業界の "暗黙知" を Claude に教え込めるのは、その業界にいた人間だけ
- 筆者の例: HRの "退職面談で出るアクションアイテムは外に出してはいけない" という業務知識を、MeetingMind AI のプロンプトに組み込んでいる

### 2.9 1年後の到達イメージ（具体年収レンジ）
- 社内DXルート: +50〜150万（評価ランクUP・AI活用手当）
- 副業ルート: +60〜360万（月5〜30万 × 12ヶ月）
- プロダクトルート: MRR $500〜$5,000（不確実だが上限が高い）
- 注意: 数字は筆者の周囲観測値であり保証ではない、と明記

### 2.10 まとめ：書けないことは弱みではない
- 2026年の希少人材は「コードを書ける人」ではなく **「現場を知っていて、AIに正しく依頼できる人」**
- 今日の夕方、自分の業務で一番イラついている10分間を Claude Code に渡してみる、から始めよう

## 3. 想定読者

- 30代前後、職種は HR・経理・営業・事務・カスタマーサクセスなどの非エンジニア
- 「AI に仕事が奪われそうで漠然と不安」だが、何から始めれば良いかわからない人
- すでに ChatGPT は使っているが、Claude Code までは踏み込めていない人
- エンジニア転職までは目指さないが、**今の職種のまま市場価値を上げたい** 人
